# Анализ энергопотребления и прогноз нагрузки в умных сетях

**Ссылка на датасет:** [London Smart Meters Dataset](https://archive.ics.uci.edu/ml/datasets/Smart+Meter+Energy+Consumption+Data+in+London)

## Этап 1. Загрузка и предварительный обзор данных

* Загрузите CSV-файл с данными энергопотребления в Pandas DataFrame.
* Проверьте структуру данных: `.info()`, `.describe()`, `.head()`.
* Составьте краткое описание признаков (дата, время, объект, потребление).
* Определите целевой признак: `consumption` (энергопотребление).

## Этап 2. Очистка, предобработка и производные признаки

* Обработайте пропуски (интерполяция или заполнение медианой/средним).
* Преобразуйте даты в формат `datetime`.
* Нормализуйте потребление для каждого объекта (Min-Max Scaling).
* Создайте ключевые производные признаки:

1. `hour_of_day` — час измерения для анализа суточных паттернов.
2. `day_of_week` — день недели для различения будней и выходных.
3. `is_weekend` — бинарный признак выходного дня (1 — выходной, 0 — будний).
4. `consumption_lag_1h` — потребление за предыдущий час для выявления трендов.
5. `rolling_std_24h` — скользящее стандартное отклонение за 24 часа для оценки изменчивости и аномалий.

## Этап 3. Разведочный анализ и проверка гипотез

| № | Гипотеза                                                    | Визуализация                                              | Тип графика                          |
| - | ----------------------------------------------------------- | --------------------------------------------------------- | ------------------------------------ |
| 1 | Суточные профили объектов различаются                       | Среднее потребление по часам для нескольких объектов      | Line Plot                            |
| 2 | Температура влияет на потребление                           | Расход энергии vs температура с полиномиальной регрессией | Scatter / Regplot                    |
| 3 | Аномалии сигнализируют о неисправностях                     | График потребления с выделенными точками за пределами ±2σ | Line Plot                            |
| 4 | Прогноз суммарной нагрузки точнее при агрегации по объектам | Сравнение прогноза Bottom-Up и Top-Down                   | Line Plot с доверительным интервалом |
| 5 | Еженедельные паттерны отличаются для будней и выходных      | Среднее потребление по дням недели                        | Small multiples                      |

* К каждому графику добавьте краткий вывод о подтверждении или опровержении гипотезы.

## Этап 4. Создание итогового отчета

* Объедините ключевые визуализации в единую фигуру с помощью `plt.subplots()`.
* Добавьте заголовки, подписи осей и пояснения к графикам.
* Отчет должен включать: цели, описание данных, визуализации, ключевые выводы.

## Этап 5. Заключение

* Подведите итоги по каждой проверенной гипотезе.
* Сформулируйте общие выводы о ключевых факторах, влияющих на энергопотребление и прогнозируемость нагрузки.
* Дайте практические рекомендации:

  * Мониторинг суточных и недельных паттернов для планирования нагрузки.
  * Обнаружение аномалий для раннего выявления неисправностей.
  * Выбор подхода к прогнозированию (Bottom-Up vs Top-Down) в зависимости от данных.
