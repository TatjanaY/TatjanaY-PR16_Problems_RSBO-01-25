# Анализ потребительского поведения и товарных категорий на данных интернет-магазина

**Ссылка на датасет:** [Brazilian E-Commerce Dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

## Этап 1. Загрузка, объединение и предварительный обзор

1. Загрузите четыре CSV-файла (`olist_orders_dataset.csv`, `olist_order_items_dataset.csv`, `olist_products_dataset.csv`, `olist_customers_dataset.csv`) с помощью Pandas.
2. Сформируйте единый аналитический `DataFrame`, объединив таблицы через ключевые поля (`order_id`, `product_id`, `customer_id`).
3. Проведите предварительный анализ объединенного набора данных:

   * Используйте `.info()` для изучения структуры и типов данных.
   * Используйте `.describe()` для получения базовой статистики.
   * Просмотрите первые строки с помощью `.head()`.
4. Составьте краткую схему данных (Data Model) с указанием связей между таблицами.

## Этап 2. Очистка, предобработка и обогащение данных

1. **Обработка пропусков и аномалий:**

   * Определите ключевые поля с пропусками и зафиксируйте их процент.
   * Преобразуйте даты (`order_purchase_timestamp`, `order_delivered_customer_date`) в тип `datetime`.
   * Удалите строки, где дата покупки позже даты доставки.
   * Проверьте поле `price` на отрицательные и нулевые значения. Замените их на медианное значение по категории товара.
2. **Создание производных признаков:**

   * Из даты покупки создайте `order_year_month`, `day_of_week`, `hour_of_day`.
   * Рассчитайте `delivery_days` – количество дней между датой доставки и покупкой; отрицательные значения и выбросы (>100 дней) замените медианой по штату покупателя.
   * Создайте `order_revenue = price * quantity` для каждой позиции заказа.

## Этап 4. Создание итогового отчета

1. Объедините ключевые визуализации из Этапа 3 в единую фигуру с помощью `plt.subplots()`.
2. Расположите графики в логической последовательности, соответствующей проверяемым гипотезам.
3. Добавьте общее название отчета и подписи к каждому графику.
