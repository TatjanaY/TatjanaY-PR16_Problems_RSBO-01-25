# 🎵 Задание: Анализ аудио-характеристик и рыночных успехов музыкальных треков

**Ссылка на датасет:** [Spotify Tracks Dataset (1986-2023)](https://www.kaggle.com/datasets/ksuqing/spotify-data-1986-2023)

**Краткое описание датасета:**
Этот обширный набор данных содержит информацию о музыкальных треках с 1986 по 2023 год, полученную через Spotify Web API. Он включает как метаданные (название трека, артист, альбом), так и детальные аудио-характеристики и коммерческие показатели.

**Колонки данных включают:**
*   **Метаданные:** `track_id`, `track_name`, `album_name`, `artists_names`, `album_release_date`, `year`, `album_type`, `explicit`.
*   **Аудио-характеристики:** `danceability`, `energy`, `valence`, `acousticness`, `instrumentalness`, `tempo`, `loudness`, `speechiness`, `liveness`, `duration_ms`, `key`, `mode`.
*   **Популярность и рынок:** `popularity` (0-100), `available_markets` (количество стран, где доступен трек).

## **Этап 2. Очистка, предобработка и создание новых признаков**
**Обработка данных:**
*   Преобразуйте колонку `album_release_date` в тип `datetime`.
*   Проверьте данные на пропуски и дубликаты, примите решение об их обработке (удаление, заполнение).
*   Исследуйте данные на наличие аномальных значений (например, нереалистичные значения длительности или темпа) и обработайте их.

**Создание производных признаков:**
*   Из `album_release_date` выделите: `release_year`, `release_month`, `season_of_release`.
*   Создайте категориальные переменные на основе числовых характеристик, например:
    *   `energy_level` (низкий, средний, высокий) на основе `energy`.
    *   `danceability_category` на основе `danceability`.
*   Рассчитайте составные показатели:
    *   `positivity_index` — комбинированный показатель на основе `valence` (позитивность) и `energy`.
    *   `complexity_score` — на основе `instrumentalness` и `speechiness` (присутствие речи).
*   Добавьте нормализованные версии ключевых аудио-характеристик (например, с помощью `MinMaxScaler`) для сравнительного анализа между треками.
*   Посчитайте средние значения аудио-характеристик по артистам (`mean_energy_by_artist`).
