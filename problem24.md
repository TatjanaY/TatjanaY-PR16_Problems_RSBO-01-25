# Анализ рынка труда и тенденций трудоустройства с использованием Python

## Цель задания
Провести полный цикл анализа данных о вакансиях и рынке труда, используя библиотеки **Python** (Pandas, NumPy, Scikit-learn, Matplotlib/Seaborn, NLTK) для обработки, анализа и визуализации. Задание фокусируется на выявлении трендов трудоустройства, кластеризации профессий и анализе требований работодателей.

## Набор данных
*   **Источники данных**: Используйте датасет **Job Dataset (8K+ Job Descriptions)** на Kaggle.
*   **Потенциальные источники**:
    *   **Основные данные**: Job Dataset (https://www.kaggle.com/datasets/mohamedharris/job-dataset)
    *   **Дополнительные данные**: US Bureau of Labor Statistics Data (BLS) через API
    *   **Дополнительные данные**: LinkedIn Job Postings Dataset (при наличии)
*   **Структура данных**:
    *   `job_descriptions`: Job_ID, Title, Company, Location, Description, Requirements, Salary_Range, Experience_Required
    *   `job_categories`: Category_ID, Category_Name, Parent_Category
    *   `skills_data`: Skill_ID, Skill_Name, Category, Demand_Trend
*   **Описание**: Данные содержат описания вакансий, требования к кандидатам, информацию о компаниях и локациях, а также данные о востребованных навыках.

## Гипотезы для проверки
1.  Средний уровень зарплат имеет выраженную географическую дифференциацию, причем различия между регионами сохраняются даже для одинаковых должностей.
2.  Требования к опыту работы в описаниях вакансий коррелируют с предлагаемой зарплатой, при этом зависимость является нелинейной (скачок зарплат после 3-5 лет опыта).
3.  Профессии могут быть эффективно сгруппированы в кластеры на основе требуемых навыков и компетенций, причем эти кластеры пересекаются с традиционными отраслевыми классификациями.
4.  Существуют устойчивые сезонные паттерны в публикации вакансий в разных отраслях (пики найма в определенные месяцы года).
5.  Частота упоминания цифровых навыков (Python, SQL, Data Analysis, AI/ML) в описаниях вакансий растет экспоненциально независимо от отрасли.

## Этапы выполнения

### Этап 1. Сбор и подготовка данных (ETL)
**Требования**:
1.  **Загрузка и очистка**:
    *   Загрузите основной датасет вакансий в `DataFrame` с помощью **Pandas**.

    *   Обработайте пропуски в числовых полях (зарплата, опыт) и категориальных данных (локация, категория).
2.  **Расчет производных показателей**:
    *   Извлеките числовые значения из полей зарплаты и опыта (например, "3-5 years" → `min_experience=3`, `max_experience=5`).
    *   Рассчитайте среднюю зарплату для каждой вакансии при наличии диапазона.
    *   Создайте бинарные признаки наличия ключевых навыков на основе анализа текста описаний.
    *   Стандартизируйте названия локаций (город, штат, страна) и категорий вакансий.
3.  **Обогащение данных**:
    *   Добавьте категоризацию профессий на основе стандартных классификаторов (например, O*NET-SOC).
    *   Рассчитайте частотность упоминаний навыков по отраслям и временным периодам.
    *   Создайте сводные таблицы по регионам, отраслям и временным промежуткам.

### Этап 3. Кластеризация профессий по навыкам (Гипотеза 3)
**Требования**:
1.  **Подготовка данных для кластеризации**:
    *   Создайте матрицу "вакансия-навык" с бинарными значениями (1 - навык упомянут, 0 - нет).
    *   Выберите топ-50 наиболее часто упоминаемых навыков для уменьшения размерности.
    *   Примените TF-IDF к текстовым описаниям вакансий для создания векторных представлений.
2.  **Кластеризация**:
    *   Объедините матрицу навыков и TF-IDF векторы.
    *   Примените алгоритм K-Means и определите оптимальное число кластеров (5-8) с помощью метода локтя и силуэтного анализа.
    *   Для сравнения примените иерархическую кластеризацию.
3.  **Анализ и визуализация**:
    *   Проанализируйте центры кластеров. Определите характерные навыки для каждого кластера и дайте им названия ("Data Science & AI", "Web Development", "DevOps & Cloud", "Business Analytics").
    *   Визуализируйте результат с помощью t-SNE для уменьшения размерности до 2D.
    *   Проверьте соответствие кластеров традиционным отраслевым классификациям.
    *   Проанализируйте распределение зарплат и требований к опыту внутри кластеров.
