# Анализ общественного транспорта и пассажиропотока в Чикаго

**Ссылка на датасет:** [Chicago Transit Authority (CTA) Ridership](https://www.kaggle.com/datasets/chicago/cta-ridership)

## Этап 1. Загрузка и предварительный обзор данных

* Загрузите CSV-файлы с данными о пассажиропотоке, станциях и погоде в Pandas DataFrame.

* Проверьте структуру данных:

  * `.info()` – типы и структура данных.
  * `.describe()` – базовая статистика числовых признаков.
  * `.head()` – первые строки для визуального осмотра.

* Составьте краткое описание каждого признака: дата, станция, количество поездок, тип линии, погодные условия и т.д.

* Определите целевые показатели: `Rides` — количество поездок на станции.

## Этап 2. Очистка, предобработка и обогащение данных

**Обработка пропусков и аномалий:**

* Проверьте наличие пропусков и зафиксируйте процент пропущенных значений.
* Обработайте отрицательные значения пассажиропотока, неверные даты.

**Создание производных признаков:**

* `Hour` – час дня из времени.
* `DayOfWeek` – день недели из даты.
* `IsWeekend` – бинарный признак: 1 — выходной/праздник, 0 — рабочий день.
* `Season` – сезон года по дате.
* `MultiLineStation` – бинарный признак: 1 — станция обслуживает >1 линии, 0 — только 1 линия.
* `ExtremeWeather` – бинарный признак: 1 — снег/дождь/шторм, 0 — нормальные условия.

**Проверка распределений:**

* Постройте гистограммы и boxplot для `Rides` для выявления выбросов.
* Проанализируйте распределение категориальных признаков (`Line_Type`, `ADA_Accessible`, `Day_Type`).

## Этап 4. Создание итогового отчета

* Объедините ключевые визуализации из Этапа 3 в единый отчет с помощью `plt.subplots()`.

* Расположите графики в логической последовательности, соответствующей проверяемым гипотезам.

* Добавьте:

  * общее название отчета;
  * подписи осей;
  * поясняющие заголовки к каждому графику.

* По желанию: визуализируйте станции на карте Чикаго с использованием `Geopandas` или `folium`.

## Этап 5. Заключение

* Подведите итоги по каждой проверенной гипотезе (подтверждена / опровергнута / частично подтверждена).
